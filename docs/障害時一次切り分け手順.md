# 障害時一次切り分け手順

最終更新日: 2026-06-25
対象: 授業担当者 / 運用担当

## 1. 症状別の初動

### 1.1 教卓画面が表示されない

1. ブラウザでトップページへアクセスできるか確認する。
2. URLが正しいか確認する。
3. ページ再読み込みを実施する。

### 1.2 進捗が更新されない

1. 課題IDが正しいか確認する。
2. 10秒以上待って更新されるか確認する。
3. ブラウザ開発者ツールで API エラー有無を確認する。

### 1.3 CSV取込で失敗する

1. ヘッダ行が `groupId,row,col,userId,studentName,label` になっているか確認する。
2. line と reason を確認し、該当行を修正する。
3. row/col が 1..100 内か確認する。
4. userId がグループメンバーか確認する。

## 2. API疎通確認コマンド

1. グループ進捗

```bash
curl -s 'http://<HOST>:<PORT>/api/dashboard/groups?problemId=A-01'
```

2. 席順進捗

```bash
curl -s 'http://<HOST>:<PORT>/api/dashboard/groups/<GROUP_ID>/seats?problemId=A-01'
```

3. 提出履歴

```bash
curl -s 'http://<HOST>:<PORT>/api/submissions?problemId=A-01'
```

## 3. ログ確認

1. サーバログに 4xx/5xx が出ていないか確認する。
2. 同時刻のアクセスログで対象エンドポイントのレスポンスを確認する。

## 4. 復旧不能時のエスカレーション

1. 発生時刻、課題ID、グループID、実施操作を記録する。
2. エラーメッセージ全文とスクリーンショットを添付する。
3. 開発担当へ連絡する。
