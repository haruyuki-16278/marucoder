### 19.1 バックエンド

#### 19.1.1 Group / GroupMember / ClassroomSeat の永続化実装

目的:
- 授業運用に必要なグループ、所属、席順データを保存できる状態にする。

実装指示:
1. Group, GroupMember, ClassroomSeat の型定義を追加する。
2. 永続化層に CRUD (作成、取得、更新、削除) を実装する。
3. groupId と userId の参照整合性を担保する。

完了条件:
1. グループ作成後に再取得できる。
2. メンバー追加と削除ができる。
3. 席順データを保存、更新、再取得できる。

成果物:
1. モデル定義ファイル
2. リポジトリ実装
3. マイグレーションまたは初期化手順

#### 19.1.2 既存 Submission に groupId を追加し、登録時に関連付け

目的:
- 提出データをグループ単位で集計可能にする。

実装指示:
1. Submission モデルへ groupId を追加する。
2. 提出作成 API で、ユーザー所属グループを解決して groupId を設定する。
3. 無所属時の扱いを仕様どおり null として保存する。

完了条件:
1. 提出レコードに groupId が保存される。
2. 既存取得 API が破壊されない。

成果物:
1. Submission スキーマ更新
2. 提出作成処理の更新

#### 19.1.3 GET /api/dashboard/groups を実装 (問題別進捗集計)

目的:
- 教卓画面の左テーブル表示データを提供する。

実装指示:
1. 入力 problemId を受け取り、グループ単位の集計を返す。
2. totalStudents, submittedStudents, acStudents, progressRate, lastSubmissionAt を算出する。
3. 0 除算回避として totalStudents が 0 の場合 progressRate を 0 にする。

完了条件:
1. 仕様書 16.1 の JSON 形式で返る。
2. データがない場合も空配列で正常応答する。

成果物:
1. API ハンドラ
2. 集計サービス

#### 19.1.4 GET /api/dashboard/groups/:groupId/seats を実装 (席順進捗)

目的:
- 教卓画面の右マップ表示データを提供する。

実装指示:
1. 指定 groupId と problemId で席順情報を返す。
2. latestVerdict, submissionCount, lastSubmittedAt を席ごとに算出する。
3. 空席は userId=null, studentName=null, latestVerdict="NONE" で返す。

完了条件:
1. 仕様書 16.2 の JSON 形式で返る。
2. row, col で安定ソートして返る。

成果物:
1. API ハンドラ
2. 席順進捗集計ロジック

#### 19.1.5 PUT /api/groups/:groupId/seats を実装 (手動更新)

目的:
- 教員が UI から席順を手動更新できる状態にする。

実装指示:
1. seats 配列の upsert を実装する。
2. 同一座標重複を検知して 400 を返す。
3. userId 指定時は group メンバー整合を検証する。

完了条件:
1. updatedCount が正しく返る。
2. 不正データ時に code/message/details を返す。

成果物:
1. API ハンドラ
2. バリデーション関数

#### 19.1.6 POST /api/groups/:groupId/seats/import を実装 (CSV取込)

目的:
- 教員が CSV 一括取り込みで席順を登録できる状態にする。

実装指示:
1. Content-Type text/csv を受け取り、17章仕様でパースする。
2. 行単位でバリデーションし、成功行のみ適用する。
3. acceptedCount, updatedCount, errorCount, errors を返す。

完了条件:
1. 正常 CSV が取り込める。
2. 異常行は行番号付きでエラー配列へ格納される。

成果物:
1. CSV パーサ実装
2. インポート API ハンドラ

#### 19.1.7 CSV バリデータ実装 (重複座標、範囲、メンバー整合)

目的:
- 席順取込品質を担保し、授業中の表示崩れを防止する。

実装指示:
1. row, col の範囲 (1..100) を検証する。
2. 同一 groupId 内の (row, col) 重複を検証する。
3. userId が group メンバーかを検証する。

完了条件:
1. ルール違反を検知し、理由文字列を返す。
2. PUT と CSV import で同一バリデータを再利用する。

成果物:
1. 共通バリデーションモジュール
